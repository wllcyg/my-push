## 一、产品概述

### 1.1 产品定位

面向全球非中文母语用户的 AI 驱动中文学习网页工具。用户无需下载 App，打开浏览器即可通过 AI 对话、汉字练习、HSK 备考三大模块系统学习中文。

### 1.2 核心差异化

| 竞品痛点 | ChineseMate 解法 |
|---|---|
| 需要下载 App | 纯网页，零门槛 |
| 英文界面为主 | UI 支持英/日/韩/西班牙语 |
| 场景固定 | 用户可自定义对话场景 |
| 只有对话无解释 | 对话同步显示拼音 + 语法注释 + 文化背景 |
| 订阅制门槛高 | 先免费跑量，后期按需付费 |

### 1.3 目标用户

- 英语母语用户（美国/英国/澳大利亚）：学习中文用于旅行、商务、文化兴趣
- 日韩用户：汉字基础好，需要口语/听力提升
- 西班牙语用户：拉美新兴学中文群体
- HSK 考生：备考 HSK 2-5 级

### 1.4 用户界面语言

MVP 阶段支持：英语、日语、韩语
后续扩展：西班牙语、法语、阿拉伯语

## 二、商业模式

### 2.1 MVP 阶段（0-3个月）

**完全免费，目标跑量**

- 无付费墙，所有功能免费
- 核心目标：获取 1000 注册用户，验证留存和使用频率
- 数据埋点：每日活跃、功能使用分布、用户流失节点

### 2.2 成长阶段（3-6个月）

根据数据决定变现模式，候选方案：

| 方案 | 适合条件 | 预期客单价 |
|---|---|---|
| 免费 + 次数包 | 用户高频使用 AI 对话 | $3.9/100次 |
| Freemium 订阅 | 用户持续每日使用 | $6.9/月 |
| 免费 + Google AdSense | 流量大但付费转化低 | 按展示收入 |

### 3.2 模块一：场景对话练习

**功能描述**

用户选择生活场景，AI 扮演对应角色，用中文与用户进行真实对话，同步提供拼音、翻译、语法注释。

**预设场景列表（MVP 共 20 个）**

| 分类 | 场景 |
|---|---|
| 日常生活 | 点咖啡、去超市、打车、问路、买药 |
| 餐饮 | 餐厅点菜、外卖电话、结账 |
| 交通 | 买火车票、机场问询、地铁换乘 |
| 商务 | 自我介绍、会议开场、商务宴请 |
| 社交 | 初次见面、找房子、邻居聊天 |
| 自定义 | 用户自由输入场景描述，AI 生成角色 |

**交互流程**

```
1. 用户选择场景（或输入自定义场景）
2. 用户选择难度：初级 / 中级 / 高级
3. AI 以角色身份发起第一句中文
4. 用户输入回复（文字，后期支持语音）
5. AI 回复，同时展示：
   - 用户输入的纠错（红字标出错误）
   - 本轮 AI 回复的拼音注释
   - 关键词翻译（hover 显示）
   - 语法点说明（可折叠）
6. 对话结束后生成学习报告：
   - 正确率
   - 本次学到的关键词
   - 推荐下一个场景
```

**AI Prompt 设计要点**

- 系统角色：严格扮演场景人物，不跳出角色
- 难度控制：初级只用 HSK1-2 词汇，高级使用成语/俚语
- 纠错逻辑：温和指出错误，给出正确表达，继续对话不中断
- 文化注释：适时插入文化背景（如"买单"的由来）

---

### 3.3 模块二：汉字 & 拼音学习

**功能描述**

系统化学习汉字的写法、读音、含义及词语搭配。

**子功能**

**① 汉字详情页**
- 汉字大图展示
- 笔顺动画（SVG 动画）
- 拼音 + 声调标注
- 部首解析
- 常用词组（3-5个）
- 例句（含拼音 + 翻译）
- 相近字对比

**② 拼音发音练习**
- 声母/韵母/声调分类练习
- 音频播放（标准普通话）
- 易混淆音对比（如 zh/z, sh/s）

**③ 汉字记忆卡片（Flashcard）**
- Anki 式间隔复习算法
- 正面：汉字，背面：拼音 + 翻译
- 用户标记"认识/不认识"
- 学习进度可视化

**④ 汉字搜索**
- 支持拼音搜索、英文搜索、部首搜索
- 搜索结果直接进入汉字详情页

---

### 3.4 模块三：HSK 备考刷题

**功能描述**

覆盖 HSK 1-6 级（新版 HSK），提供词汇练习、阅读理解、听力模拟（文字版）。

**子功能**

**① 词汇练习**
- 按 HSK 级别分类
- 四选一选择题
- 填空题
- 错题本自动收录

**② 阅读理解**
- 短文 + 问题形式
- AI 生成的仿真题
- 答题后 AI 解析每道题的答题思路

**③ 模拟测试**
- 按 HSK 真题格式出题
- 计时功能
- 成绩分析报告（各题型正确率）

**④ 学习计划**
- 用户输入目标级别 + 考试日期
- AI 生成每日学习任务
- 进度追踪

---

### 3.5 用户系统

**注册/登录**
- Google OAuth（主要，面向国际用户）
- 邮箱注册
- GitHub OAuth（可选）

**个人主页**
- 学习天数连续记录（streak）
- 各模块累计学习时长

**学习记录**
- 历史对话记录（可回顾）
- 错题本
- 收藏的词汇

---

### 3.6 多语言 UI

- 语言切换：页面右上角下拉选择
- MVP 支持：英语（默认）、日语、韩语
- 实现方式：i18n 静态文案翻译，AI 回复语言跟随 UI 语言自动切换

## 四、技术架构

### 4.1 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 前端 | Next.js 14 + TypeScript | App Router，SSR 利于 SEO |
| 样式 | Tailwind CSS | 快速开发 |
| 数据库 | Supabase (PostgreSQL) | 用户数据、学习记录 |
| 认证 | Supabase Auth | Google OAuth + 邮箱 |
| AI | Claudfire (Claude API 中转) | 对话、纠错、题目生成 |
| 部署 | Vercel | 免费套餐够用，全球 CDN |
| 域名 | 自购 .com 域名 | 建议：chinesemate.com 类 |

### 4.2 数据库设计（核心表）

```sql
-- 用户表
users (id, email, name, avatar, ui_language, created_at)

-- 学习记录表
learning_sessions (id, user_id, module, scene, duration, score, created_at)

-- 汉字掌握表
user_characters (id, user_id, character, status, next_review_at)

-- 错题本
wrong_answers (id, user_id, question_id, user_answer, correct_answer, created_at)

-- 对话历史
conversations (id, user_id, scene, messages jsonb, created_at)
```

### 4.3 AI 调用策略

- 场景对话：流式输出（streaming），减少等待感
- 纠错分析：单独调用，结构化 JSON 输出
- 题目生成：批量生成后缓存，减少 API 消耗
- 每用户每日免费额度：对话 20 轮、刷题 50 题

## 八、风险与应对

| 风险 | 可能性 | 应对 |
|---|---|---|
| AI API 成本超预期 | 中 | 设置每用户每日使用上限，超出提示注册 |
| 竞品打价格战 | 低 | 差异化在网页免下载 + 多语言，不打价格 |
| SEO 见效慢 | 高 | 同步做 Reddit/社区运营，不单靠 SEO |
| 用户注册但不留存 | 中 | 每日学习 streak 机制 + 邮件提醒 |
| Vercel 免费套餐不够 | 低 | 初期流量有限，够用；超了说明成功了 |
