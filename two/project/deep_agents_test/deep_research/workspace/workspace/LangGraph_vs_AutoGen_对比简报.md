# LangGraph vs AutoGen 框架对比分析简报

> 生成日期：2025年 | 版本：v1.0

---

## 一、执行摘要

LangGraph 与 AutoGen 是当前 AI Agent 领域最具代表性的两大开源框架，分别由 **LangChain Inc.** 和 **Microsoft Research** 开发。两者虽都面向 LLM 驱动的多智能体应用，但在**核心架构、设计哲学、适用场景**上存在根本性差异：

- **LangGraph** = **图驱动的状态机**，以"有向图 + 持久化状态"为核心，强调对执行流程的精细控制，适合构建**生产级、可恢复的复杂工作流**。
- **AutoGen** = **对话驱动的多智能体协作**，以"Agent 间对话"为核心，强调多角色的自然协作与代码执行，适合构建**灵活的多角色协作系统**。

## 三、核心架构对比

### 3.1 LangGraph 架构：图 + 状态 + 检查点

```
┌──────────────────────────────────────────────┐
│               LangGraph 架构                  │
├──────────────────────────────────────────────┤
│  State（共享状态对象）                          │
│  ├── TypedDict / Pydantic 定义                │
│  ├── Reducer 控制合并策略                      │
│  └── 贯穿全图的单一状态流                       │
├──────────────────────────────────────────────┤
│  Graph（有向图）                               │
│  ├── Node（计算单元：函数/LLM/工具）            │
│  ├── Edge（流转路径：普通边/条件边）             │
│  ├── 支持循环、分支、条件路由                    │
│  └── compile() 编译为可执行对象                 │
├──────────────────────────────────────────────┤
│  Persistence（持久化层）                       │
│  ├── Checkpointing（每步状态快照）              │
│  ├── Thread（会话隔离）                        │
│  ├── Time Travel（回溯历史状态）                │
│  └── Store（跨会话记忆）                       │
├──────────────────────────────────────────────┤
│  Control（控制能力）                           │
│  ├── Human-in-the-Loop（中断/恢复）            │
│  ├── Retry（节点级重试）                       │
│  └── Streaming（流式输出）                     │
└──────────────────────────────────────────────┘
```

**核心思想**：将 AI 应用建模为一个**有状态的有向图**，每个节点执行计算，状态在节点间流转和更新，支持循环、条件分支和持久化。

### 3.2 AutoGen 架构：Agent + 对话 + 团队

```
┌──────────────────────────────────────────────┐
│               AutoGen 架构（v0.4）            │
├──────────────────────────────────────────────┤
│  autogen-agentchat（应用层）                   │
│  ├── AssistantAgent（LLM 驱动的助手）          │
│  ├── Teams（RoundRobin / Selector GroupChat） │
│  └── 预构建的多 Agent 协作模式                  │
├──────────────────────────────────────────────┤
│  autogen-core（核心层）                        │
│  ├── 异步消息传递基础设施                       │
│  ├── Agent 运行时环境                          │
│  ├── 事件驱动架构                              │
│  └── 可观测性（OpenTelemetry）                 │
├──────────────────────────────────────────────┤
│  autogen-ext（扩展层）                         │
│  ├── 模型客户端（OpenAI/Azure/Ollama）         │
│  ├── 工具集成                                  │
│  └── 代码执行器（Docker 沙箱）                  │
├──────────────────────────────────────────────┤
│  Conversation Patterns（对话模式）              │
│  ├── PairChat（两两对话）                      │
│  ├── GroupChat（群组对话）                     │
│  ├── Nested Chat（嵌套对话）                   │
│  └── Human-in-the-Loop（人类代理）              │
└──────────────────────────────────────────────┘
```

**核心思想**：将多智能体协作建模为**对话过程**，Agent 通过消息交换完成任务，支持多种对话模式和人类自然介入。

## 四、设计哲学对比

| 维度 | LangGraph | AutoGen |
|------|-----------|---------|
| **核心隐喻** | 流程图 / 状态机 | 群聊 / 会议 |
| **控制范式** | **显式控制**：开发者精确定义每个节点和边 | **隐式涌现**：Agent 通过对话自主协作 |
| **状态管理** | 中央化状态对象，全局可见可追踪 | 分布式对话历史，各 Agent 维护自身上下文 |
| **流程定义** | 图结构预定义，条件边动态路由 | 对话模式预定义，发言顺序可动态选择 |
| **执行模型** | 状态机驱动，支持暂停/恢复/回溯 | 对话轮次驱动，支持终止条件控制 |
| **抽象层级** | 低层级编排（接近"汇编"） | 中高层编排（接近"对话编排"） |
| **开发者角色** | 流程架构师：设计图的拓扑和状态流转 | 团队教练：配置 Agent 角色和对话规则 |

### 关键差异解读

**LangGraph 的"显式控制"哲学**：
- 开发者像画流程图一样定义整个工作流
- 每个节点做什么、何时跳转、如何分支都由代码精确控制
- 适合需要**确定性、可审计、可恢复**的生产场景

**AutoGen 的"对话涌现"哲学**：
- 开发者定义 Agent 角色和对话规则，具体协作过程由 LLM 驱动
- Agent 之间可以辩论、纠错、迭代，协作过程更灵活
- 适合需要**灵活性、创造性、多角色协作**的探索场景

## 五、关键能力对比

### 5.1 状态管理与持久化

| 能力 | LangGraph | AutoGen |
|------|-----------|---------|
| 状态持久化 | ✅ 内置 Checkpointing，每步自动保存 | ❌ 无内置持久化，需自行实现 |
| 状态回溯 | ✅ Time Travel，可回溯到任意历史状态 | ❌ 不支持 |
| 跨会话记忆 | ✅ Store 机制，支持语义查询 | ⚠️ v0.4 引入 Memory 概念，尚在发展 |
| 会话隔离 | ✅ Thread 机制，天然隔离 | ⚠️ 需自行管理 |
| 断点续传 | ✅ 从任意检查点恢复执行 | ❌ 不支持 |

**结论**：LangGraph 在状态管理上具有压倒性优势，这是其作为"生产级运行时"的核心竞争力。

### 5.2 多 Agent 协作

| 能力 | LangGraph | AutoGen |
|------|-----------|---------|
| 多 Agent 支持 | ✅ 通过多节点/子图实现 | ✅ 核心能力，原生支持 |
| 协作模式 | 图结构定义（灵活但需手动设计） | 对话模式（PairChat/GroupChat/Nested） |
| Agent 角色定义 | 通过节点函数 + 系统提示词 | 通过 Agent 类型 + 系统提示词 |
| 发言调度 | 条件边动态路由 | GroupChatManager 多种策略 |
| Agent 间通信 | 共享状态对象 | 消息传递（对话历史） |
| 嵌套协作 | ✅ 子图（Subgraph）机制 | ✅ Nested Chat 机制 |

**结论**：AutoGen 在多 Agent 对话协作上更加自然和直观，LangGraph 则提供更精细的流程控制。

### 5.3 代码执行能力

| 能力 | LangGraph | AutoGen |
|------|-----------|---------|
| 内置代码执行 | ❌ 需自行集成 | ✅ 原生支持 |
| 沙箱执行 | ❌ 需自行实现 | ✅ Docker 容器隔离 |
| 代码生成-执行循环 | 需手动设计图结构 | ✅ 内置 Agent 间自动循环 |
| 安全性 | 取决于实现 | Docker 隔离 + 超时控制 |

**结论**：AutoGen 在代码执行方面具有显著优势，开箱即用。

### 5.4 人机协作（Human-in-the-Loop）

| 能力 | LangGraph | AutoGen |
|------|-----------|---------|
| 中断/恢复 | ✅ interrupt 机制 + 检查点持久化 | ⚠️ UserProxyAgent 模式 |
| 人工审批 | ✅ 任意节点可设置中断点 | ✅ 代码执行前可设置审批 |
| 渐进式自动化 | ⚠️ 需手动设计 | ✅ human_input_mode 三档配置 |
| 状态保持 | ✅ 中断后状态完整保存 | ❌ 中断后需重新建立对话 |

**结论**：LangGraph 的人机协作更加健壮（有状态持久化保障），AutoGen 的配置更加简便。

### 5.5 可观测性与调试

| 能力 | LangGraph | AutoGen |
|------|-----------|---------|
| 执行追踪 | ✅ LangSmith 集成 | ✅ OpenTelemetry 支持 |
| 可视化调试 | ✅ LangGraph Studio | ⚠️ AutoGen Studio（GUI） |
| 消息追踪 | ✅ 完整的状态变更历史 | ✅ 消息历史记录 |
| 性能监控 | ✅ LangSmith 平台 | ✅ OpenTelemetry 生态 |

### 5.6 可扩展性与部署

| 能力 | LangGraph | AutoGen |
|------|-----------|---------|
| 分布式部署 | ✅ LangGraph Platform | ✅ v0.4 原生支持分布式 Agent |
| 异步执行 | ✅ 支持 | ✅ v0.4 全面异步化 |
| 跨语言 | Python / JS | Python / .NET |
| 商业化支持 | ✅ LangGraph Platform（商业产品） | ⚠️ 社区驱动 |

## 六、适用场景对比

### 6.1 LangGraph 最适合的场景

| 场景 | 原因 |
|------|------|
| **生产级 Agent 系统** | 检查点 + 断点续传 + 容错，确保系统可靠性 |
| **长时运行工作流** | 状态持久化支持跨小时/天/周的流程 |
| **需要审计追踪的系统** | 完整的状态变更历史，满足合规要求 |
| **复杂条件逻辑工作流** | 图结构天然支持复杂分支、循环、并行 |
| **多轮对话 + 工具调用** | ReAct 模式的循环 + 状态管理 |
| **需要人工审批的流程** | 中断/恢复 + 状态保持 |
| **RAG 增强应用** | 多步检索-推理-验证的复杂流程 |
| **需要回溯和调试的系统** | Time Travel 能力 |

**典型用户**：需要构建**可靠、可控、可审计**的 AI 应用的企业开发团队。

**代表企业用户**：LinkedIn、Uber、Klarna、GitLab

### 6.2 AutoGen 最适合的场景

| 场景 | 原因 |
|------|------|
| **多角色协作任务** | 群聊模式天然支持多专家角色协作 |
| **代码生成与数据分析** | 内置代码执行 + 生成-执行-反馈循环 |
| **快速原型开发** | 对话驱动，配置简单，上手快 |
| **创意/探索性任务** | Agent 间辩论、纠错，适合开放性任务 |
| **软件开发模拟** | 产品经理→架构师→工程师→审查员的角色链 |
| **教育与研究** | 苏格拉底式对话、多 Agent 辩论 |
| **需要代码沙箱的场景** | Docker 隔离执行，安全可控 |
| **渐进式人机协作** | 三档 human_input_mode 配置 |

**典型用户**：需要快速构建**多角色协作、代码密集型** AI 应用的研究者和开发者。

**代表应用**：AutoGen Studio（可视化多 Agent 构建工具）

## 七、优劣势总结

### 7.1 LangGraph

| ✅ 优势 | ❌ 劣势 |
|---------|---------|
| 精细的流程控制（图结构） | 学习曲线陡峭 |
| 强大的状态持久化与检查点 | 简单场景过度设计 |
| 断点续传与容错能力 | 调试复杂度较高 |
| Time Travel 回溯能力 | 性能开销（持久化） |
| 生产就绪（企业验证） | API 变化频繁 |
| 完整的生态工具链 | 依赖 LangChain 生态 |
| 商业化支持（LangGraph Platform） | 多 Agent 对话不如 AutoGen 直观 |

### 7.2 AutoGen

| ✅ 优势 | ❌ 劣势 |
|---------|---------|
| 对话驱动，直观易懂 | v0.2 架构限制（已重写） |
| 内置代码执行能力 | 无内置状态持久化 |
| 灵活的多角色协作 | 多轮对话 LLM 成本高 |
| 快速原型开发 | 复杂流程控制不如 LangGraph |
| 渐进式人机协作 | 安全性需仔细设计 |
| 微软背景，社区活跃 | v0.4 不向后兼容 |
| 跨语言支持（Python + .NET） | 生产级部署案例较少 |

## 八、技术选型决策树

```
你的需求是什么？
│
├─ 需要生产级可靠性（断点续传、状态持久化、容错）？
│  └─ ✅ 选 LangGraph
│
├─ 需要复杂的工作流控制（条件分支、循环、并行）？
│  └─ ✅ 选 LangGraph
│
├─ 需要审计追踪和合规性？
│  └─ ✅ 选 LangGraph
│
├─ 需要快速构建多角色协作原型？
│  └─ ✅ 选 AutoGen
│
├─ 核心任务是代码生成/数据分析？
│  └─ ✅ 选 AutoGen
│
├─ 需要 Agent 间辩论/纠错的灵活协作？
│  └─ ✅ 选 AutoGen
│
├─ 两者都需要？
│  └─ 🔀 混合使用：LangGraph 做顶层编排 + AutoGen Agent 作为节点
│
└─ 不确定？
   └─ 📌 建议：生产系统选 LangGraph，研究/原型选 AutoGen
```

## 九、混合使用方案

两个框架并非互斥，可以组合使用：

```
┌─────────────────────────────────────────────┐
│            LangGraph（顶层编排）               │
│                                              │
│  [Node A] ──→ [Node B] ──→ [Node C] ──→ END │
│                  │                           │
│                  ▼                           │
│          ┌──────────────┐                    │
│          │  AutoGen      │                    │
│          │  GroupChat    │                    │
│          │  (子图节点)    │                    │
│          │  Agent1↔Agent2│                    │
│          └──────────────┘                    │
└─────────────────────────────────────────────┘
```

- **LangGraph** 负责：整体流程编排、状态持久化、错误恢复、人机审批
- **AutoGen** 负责：特定节点内的多 Agent 对话协作、代码生成与执行

## 十、总结建议

| 决策因素 | 推荐选择 |
|----------|----------|
| 生产级部署 | **LangGraph** |
| 快速原型 | **AutoGen** |
| 复杂工作流 | **LangGraph** |
| 多角色协作 | **AutoGen** |
| 代码执行 | **AutoGen** |
| 状态持久化 | **LangGraph** |
| 企业合规 | **LangGraph** |
| 研究探索 | **AutoGen** |
| .NET 生态 | **AutoGen** |
| 商业支持 | **LangGraph** |

**一句话总结**：LangGraph 是"精密的瑞士手表"——精确、可靠、可恢复；AutoGen 是"高效的头脑风暴会"——灵活、协作、富有创造力。选择取决于你是需要**精确控制流程**还是**激发多角色协作**。
