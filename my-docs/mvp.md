# 证件照工坊 — 产品需求文档(PRD)

**版本**: v0.1 draft
**平台**: HarmonyOS NEXT(纯血鸿蒙)
**日期**: 2026-07

---

## 1. 背景与机会

### 1.1 市场缺口
- 鸿蒙原生应用生态目前处于"设备多、应用少"的阶段(4200万+ 装机量,原生应用约1.5万款),证件照这个具体细分场景**没有成熟的深度产品**。
- 已上架的"白描 PDF"覆盖了扫描/OCR/PDF转换的大盘子,证件照只是其中一个附属小功能,不是主打场景,没有覆盖到证件照场景真正的核心痛点(精确像素/文件体积合规、多场景尺寸库)。
- 证件照是**高频刚需、低替代性**的场景:入学、考试报名、签证、驾照、职业资格证、社保、招聘网站……几乎每个成年人每年都会用到 1-3 次,且经常在"临时需要、马上要交"的场景下发生,付费意愿强。

### 1.2 为什么适合做成鸿蒙原生 App(而不是小程序/网页)
- 抠图、换底色、压缩这些操作全部可以**离线完成**,不需要服务器,零运维成本。
- 华为系统自带 **Core Vision Kit**(人像分割/人脸检测)可以直接调用,不需要自己训练部署 AI 模型,大幅降低开发成本。
- "证件照涉及身份证件照片,处理过程不上传云端"是天然的信任卖点,尤其对中老年用户、体制内考试用户群体有很强说服力。

### 1.3 应对鸿蒙审核 3.5 条款(功能独特性)
鸿蒙审核对"功能单一、缺乏实质服务"的应用有明确驳回倾向,因此本产品**不做成单一功能的"换底色工具"**,而是围绕"证件照制作全流程"设计一组强关联功能形成完整服务闭环(详见第 4 节)。

| 职场人士 | 工作证、内部系统头像 | 批量导出多规格 |

## 3. 竞品简析

| 产品 | 定位 | 与本产品的差异 |
|---|---|---|
| 白描 PDF(鸿蒙版) | 文档扫描/OCR/PDF工具,证件照是附属功能 | 证件照场景做得浅,尺寸库不全,非核心卖点 |
| 天天P图/美图证件照(安卓/iOS) | 泛修图 App 的证件照子功能 | 未上架鸿蒙,且偏"美颜"而非"合规工具" |
| 各类考试官方系统自带裁图工具 | 仅限特定考试使用 | 不通用,体验差,常见"上传失败"投诉 |

**差异化定位一句话**:不是修图 App,是"证件照合规工具"——核心价值是**尺寸和文件大小精确匹配官方要求**,美颜是锦上添花不是主打。

## 4. 核心功能模块(MVP 范围)

为满足审核对"功能完整性"的要求,MVP 包含以下 5 个强关联模块,形成"选场景 → 拍/传照片 → 智能处理 → 合规校验 → 导出/打印"的完整闭环:

### 4.1 场景化尺寸库(核心差异点)
- 内置国家标准 + 常见考试/证件的尺寸数据库,按分类检索:
  - 身份证件类:一代/二代身份证、护照、港澳通行证、驾驶证
  - 考试类:教师资格证、计算机等级考试、公务员考试、研究生考试、四六级
  - 签证类:美签、日签、申根签等主要国家标准
  - 求职类:简历照、工作证
- 每条记录包含:像素尺寸、mm物理尺寸、文件大小要求(KB区间)、背景色要求、DPI
- 支持关键词搜索(如输入"教师资格证"直接匹配对应规格)

### 4.2 智能抠图 + 换底色
- 调用系统 Core Vision Kit 人像分割能力,自动抠出人像
- 一键替换为标准底色:白/红/蓝(国标证件照三色)
- 人脸检测辅助自动居中、按照标准比例裁剪(眼睛位置、头顶留白比例符合规范)

### 4.3 文件大小精确压缩(高痛点、差异化)
- 很多报名系统要求"文件大小需在 20KB-200KB 之间",用户常年被这个要求卡住反复尝试
- 本产品支持**指定目标文件大小区间**,自动通过质量/尺寸调整迭代压缩至符合要求,而不是让用户自己试错调整压缩率

### 4.4 排版打印
- 支持将证件照按照冲印规格排版(如六寸相纸排 8 张一寸/6 张二寸),生成可直接送冲印店打印的排版图
- 常见于线下办事仍需要纸质照片的场景(如部分政务窗口)

### 4.5 批量导出
- 一次拍摄/上传,勾选多个尺寸规格,一键批量生成全部规格的成片,分别导出

## 5. 非功能需求

- **全离线处理**:所有图像处理在本地完成,不上传服务器,App 权限说明中明确标注
- **不强制注册登录**:降低使用门槛,首次打开即可用
- **性能**:单张处理(抠图+换底色+裁剪)应在 2 秒内完成
- **隐私合规**:需在应用市场审核提交时清晰说明本地处理机制,避免被判定为"涉及敏感证件照片"额外审查

## 6. 技术方案概要

| 模块 | 技术方案 |
|---|---|
| UI 框架 | ArkTS + ArkUI(声明式) |
| 人像分割/人脸检测 | Core Vision Kit(系统内置,免自建模型) |
| 图像编解码/变换/压缩 | Image Kit |
| 尺寸库数据 | 本地 JSON 静态数据 + Preferences 做用户收藏 |
| 文件读写/导出 | fileIo + DocumentViewPicker |
| 打印排版渲染 | Canvas 自绘 |

> 备注:该技术组合完全不依赖 MindSpore Lite 自训练模型,人像分割和人脸关键点检测都走系统能力,大幅降低你在 AI 模型转换/调优上的踩坑成本(参考此前 ImageToolbox 分析中提到的 NPU 算子兼容性问题——这里直接绕开了)。

## 7. 商业化模式

- **基础版免费**:常用尺寸(身份证、简历照、护照)+ 基础换底色 + 压缩,永久免费,保证冷启动获客
- **订阅解锁**(参考白描定价区间,月付 6-12 元 / 年付 30-50 元):
  - 完整尺寸库(考试类、签证类全量)
  - 批量导出无限制(免费版限制每次导出张数)
  - 排版打印模板全解锁
  - 去除处理过程中的水印(如有)
- **不做广告变现**:证件照场景用户对"隐私+专业感"敏感,插广告会破坏信任感,不符合定位

## 8. 里程碑规划

| 阶段 | 内容 | 预估周期 |
|---|---|---|
| Phase 1(MVP) | 尺寸库(覆盖前20个最高频场景)+ 抠图换底色 + 尺寸裁剪 + 文件大小压缩 | 3-4 周 |
| Phase 2 | 批量导出 + 排版打印 | 1-2 周 |
| Phase 3 | 订阅体系接入 + 尺寸库扩充 + 用户反馈迭代 | 持续 |
| Phase 4(可选) | 折叠屏/平板适配,冲上华为"星光大道"推荐 | 视 Phase 1-3 数据决定 |

## 9. 风险与应对

| 风险 | 应对方案 |
|---|---|
| 审核 3.5 条款仍判定功能单薄 | 提交时突出"尺寸数据库+合规校验"的独特服务属性,而非单纯修图工具;可参考白描的应用描述话术 |
| Core Vision Kit 人像分割精度不足(复杂背景) | MVP 阶段先支持"纯色背景"高精度场景,复杂背景标注为"Beta"逐步优化 |
| 尺寸库数据准确性(官方标准更新) | 建立可远程更新的尺寸库(唯一联网需求点),避免因过时数据导致用户投诉 |
| 冷启动获客难 | 上线时主打"考公考编报名季"节点投放,配合华为应用市场编辑推荐位申请 |

---

## 10. 成功指标(北极星指标)

- MVP 上线 90 天内:安装量 5,000+,应用市场评分 ≥ 4.5
- 付费转化率:参考同类工具类应用鸿蒙市场基准(5%+)
- 次月留存:因场景本身低频(非日活类工具),留存指标改为"半年内二次使用率" ≥ 30%
