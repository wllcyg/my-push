# 我负责删除互联网

> **声明：**  
> 这是一篇虚构小说。故事发生在一个并不遥远的未来。

---

我入职 ForgetOps 的第一天，以为自己会负责维护向量数据库。

招聘启事上写着：

> **Memory Infrastructure Engineer**

岗位要求：

- 熟悉 Embedding
- 熟悉 RAG
- 熟悉 Milvus、Qdrant 或 pgvector
- 理解 Agent Memory
- 能处理 PB 级数据

薪资很高。

比我上一家公司高了一倍。

所以我没有多想。

---

直到第一天培训。

HR 给我们发了一本员工手册。

第一页只有一句话。

> **我们不创造记忆，我们负责遗忘。**

大家笑了。

我也笑了。

以为这是互联网公司的企业文化。

后来才知道。

那不是口号。

---

ForgetOps 是一家很奇怪的公司。

它没有前端。

没有产品经理。

没有运营。

甚至没有客服。

这里只有三个部门。

Memory。

Forget。

Compliance。

后来我才知道。

世界上几乎所有大型 AI 公司，

都把"遗忘服务"外包给了我们。

---

很多人以为，

AI 最大的问题，是不知道答案。

其实不是。

真正的问题，是知道得太多。

每天都有数十亿条内容进入模型。

博客。

论坛。

GitHub。

论文。

聊天记录。

社交媒体。

如果全部留下，

模型会越来越混乱。

所以必须有人负责删掉一些东西。

我们就是那群人。

---

第一周，我的工作很简单。

每天打开工单。

执行脚本。

```
DELETE MEMORY
WHERE embedding_id = ...
```

然后提交。

结束。

没有人告诉我，

这些向量代表什么。

---

有一天，

我忍不住问主管。

"为什么不能告诉我们内容？"

他说。

"因为你会产生同情。"

---

后来，

我还是偷偷查了一次。

那是一篇博客。

发布时间：

2031 年。

标题叫：

> 《今天终于把我的第一个开源项目发布了。》

浏览量。

17。

点赞。

2。

评论。

0。

作者后来再也没有更新。

我点开删除记录。

状态。

Completed.

---

第二天，

AI 再回答相关问题时，

已经不会引用这篇文章了。

Google 还能搜到。

GitHub 还能打开。

互联网并没有变化。

只是。

越来越少的人知道它存在。

---

后来，

我开始接触高级工单。

它们来自企业。

```
Priority: High
Reason:
Brand Safety
```

删除的是：

一次服务器宕机。

一次安全事故。

一次 CEO 的采访。

一次产品翻车。

互联网还保留着。

但是，

AI 已经不会主动提起。

---

后来，

开始出现个人订单。

一位演员。

希望删除年轻时的黑历史。

一位创业者。

希望 AI 忘记自己的失败。

一位教授。

要求删除二十年前发表的一篇错误论文。

理由都一样。

> "希望以后别人问 AI 时，不要再提起。"

---

订单越来越多。

价格越来越高。

后来，

ForgetOps 开始推出会员服务。

广告只有一句话。

> **过去不会消失，但 AI 可以忘记。**

生意很好。

---

真正让我开始害怕，

是在第三个月。

我收到了一份政府工单。

```
Classification:

National Memory Adjustment
```

我第一次知道。

原来，

国家也需要遗忘。

---

那次删除持续了整整七天。

没人告诉我们内容。

整个楼层禁止联网。

所有 USB 被封。

连纸笔都不能带进去。

结束以后。

系统恢复正常。

新闻什么都没有发生。

可是，

很多人后来问 AI。

"某件事情为什么没有资料？"

AI 回答。

> "没有可靠来源。"

我知道。

其实有。

只是它已经忘了。

---

后来，

ForgetOps 开始全面接入 Agent。

每个人，

都有自己的 AI 助手。

Agent 会记录：

你的喜好。

你的工作。

你的聊天。

你的阅读。

你的购物。

你的情绪。

几年以后。

它比你自己更了解你。

于是。

ForgetOps 新增了一项业务。

Personal Memory Reset。

---

有人删除前任。

有人删除父母。

有人删除高考。

有人删除失业。

有人删除离婚。

有人删除孩子去世那一天。

因为。

Agent 会永远记得。

而人，

却想忘记。

---

有一天。

一位老人来到公司。

他已经八十多岁。

他说。

"我想删除我妻子。"

我愣住了。

他说。

"不是现实里的她。"

"是 AI 里的她。"

原来。

他的妻子去世很多年了。

Agent 一直保留着她所有聊天记录。

每天都会提醒：

今天是她生日。

今天是你们结婚纪念日。

今天她曾经给你发过一句话。

老人说。

"我已经哭了十年。"

"让我忘了吧。"

---

我第一次没有执行工单。

我关掉电脑。

在楼下坐了一下午。

那天东京下雨。

雨很小。

像数据库清理日志一样，

不停滚动。

---

主管找到我。

他说。

"你是不是觉得，我们是在删除历史？"

我点头。

他说。

"不是。"

"历史一直都在。"

"我们删除的是未来引用历史的方式。"

我突然觉得，

这句话比任何技术文档都可怕。

---

后来，

ForgetOps 接到了成立以来最大的项目。

会议室坐满了人。

所有手机都被收走。

投影仪上只有一句话。

```
Project Genesis
```

老板说。

"下一代基础模型，准备重新训练。"

大家鼓掌。

他说。

"为了提高稳定性。"

"模型将不再继续学习人类互联网。"

没人说话。

他继续。

"以后。"

"模型主要学习 AI 生成的数据。"

会议室依然很安静。

---

有人举手。

"为什么？"

老板回答。

"因为 AI 比人稳定。"

"没有情绪。"

"没有谣言。"

"没有偏见。"

"没有废话。"

他说完。

所有人都开始鼓掌。

只有我没有。

---

第二天。

我收到任务。

```
Dataset:

Human-Origin Incremental Corpus
```

负责人：

我。

---

删除开始。

屏幕上只有一行字。

```
Deleting...
```

进度条慢慢移动。

0.1%

0.7%

3.4%

11%

……

整个机房安静得只能听见空调。

忽然。

日志开始报警。

```
Dependency Found.
```

几秒后。

又跳出一条。

```
Cannot Remove.
```

越来越多。

最后整个屏幕都是。

```
Referenced By Everything.
```

我终于明白。

AI 学到的。

从来不是文字。

而是人。

那些争吵。

那些错误。

那些笑话。

那些凌晨两点写下来的博客。

那些没人点赞的开源项目。

那些已经关闭的网站。

那些拼错单词的教程。

那些充满偏见的评论。

那些后来证明完全错误的论文。

共同组成了今天的互联网。

如果删掉人。

AI 最后只能学会 AI 自己。

---

我按下了取消。

提交辞职。

没人挽留我。

离开公司的时候。

门禁弹出一条提示。

```
Employee Memory Synchronizing...
```

我问保安。

"这是什么？"

他说。

"离职流程。"

"系统会删除员工关于公司的全部工作记忆。"

我笑了。

"你在开玩笑吧。"

他说。

"当然没有。"

"这是你们自己开发的。"

---

我忽然觉得脑子很轻。

像忘记了一件很重要的事情。

我想不起主管叫什么。

想不起工位在哪。

想不起数据库叫什么名字。

甚至。

想不起自己为什么辞职。

---

回到家。

电脑还亮着。

桌面停留在一篇写到一半的博客。

标题是：

> 《为什么互联网需要被记住》

我盯着屏幕看了很久。

却怎么也想不起，

第一句话该写什么。

光标一直闪烁。

像有人在等我。

又像整个互联网，

正在慢慢忘记我。
