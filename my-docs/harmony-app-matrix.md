# 鸿蒙工具矩阵 — 产品规划文档

**策略版本**: v1.1（工具集合模式）
**更新日期**: 2026-07

---

## 核心策略

### 为什么不做独立小工具

华为应用市场审核 3.5 条款明确驳回"功能单一、缺乏实质服务"的 App。
单独上架"图片压缩"、"格式转换"这类单功能工具，大概率被拒。

### 工具集合模式的优势

| 维度 | 20个独立小工具 | 5个工具集合 |
|---|---|---|
| 过审概率 | ~30%（大多数被拒） | ~90% |
| 用户价值 | 低（装多个 App 很麻烦） | 高（一个 App 解决多个问题） |
| 付费转化 | 低（单功能不值得付费） | 高（功能多更容易买账）|
| 维护成本 | 高（分散维护） | 低（集中迭代） |
| 代码复用率 | 低（重复搭框架） | 高（共用底层库） |

## 5 个工具集合 App 规划

---

### App 1：证件照工坊（已规划 MVP）

**定位**：证件照制作全流程工具，"合规"是核心卖点

| 功能模块 | 说明 |
|---|---|
| 场景化尺寸库 | 身份证/护照/签证/考试/简历照，20+ 高频规格 |
| 智能抠图换底色 | Core Vision Kit 人像分割，白/红/蓝一键切换 |
| 文件大小精确压缩 | 自动迭代压缩到指定 KB 区间 |
| 排版打印 | 六寸相纸排版，可直接送冲印店 |
| 批量导出 | 一次上传，多规格同时生成 |

**技术核心**：Core Vision Kit + Image Kit + Canvas
**上线周期**：Phase 1（6周）
**商业模式**：基础免费 + 订阅解锁完整尺寸库

---

### App 2：图像工具箱

**定位**：日常图片处理全能工具，替代多个分散的单功能 App

| 功能模块 | 说明 |
|---|---|
| 图片压缩 | 指定目标文件大小，自动迭代压缩 |
| 格式转换 | JPG / PNG / WEBP 互转 |
| 图片裁剪 | 自定义比例，常用比例快捷选 |
| 水印添加 | 文字/图片水印，自定义位置透明度 |
| 图片拼接 | 多图垂直/横向拼接，九宫格布局 |
| 去除背景 | 批量抠图，Core Vision Kit 人像分割 |

**技术核心**：Image Kit + Canvas + Core Vision Kit
**代码复用率**：与证件照工坊共用 70% 底层代码
**上线周期**：证件照工坊上线后 1-2 周

---

### App 3：头像设计工坊

**定位**：社交/职场头像的专项制作工具

| 功能模块 | 说明 |
|---|---|
| 智能头像裁剪 | Core Vision Kit 人脸检测自动居中 |
| 形状裁剪 | 圆形/方形/圆角/自定义形状 |
| 背景替换 | 换纯色/渐变/模糊背景 |
| 边框/装饰 | 节日头像框、职业认证框 |
| 多平台适配 | 微信/微博/LinkedIn/GitHub 专用尺寸 |
| 简历头像 | 职场标准头像比例，一键生成 |

**技术核心**：Core Vision Kit 人脸检测 + Canvas
**代码复用率**：与证件照工坊共用 60% 代码
**上线周期**：证件照工坊上线后 2-3 周

---

### App 4：办公文件助手

**定位**：个人轻量办公场景，取代 PC 端的繁琐操作

| 功能模块 | 说明 |
|---|---|
| 电子签名 | 手写签名生成透明 PNG，可叠加到文档截图上 |
| 电子印章 | 输入文字生成圆形红章图片 |
| 证书/奖状生成 | 内置模板，填写信息一键生成可打印图片 |
| 名片制作 | 多套模板，填写信息生成名片图 |
| 图片加文字 | 在图片上添加标注/说明文字 |

**技术核心**：Canvas 自绘 + fileIo
**代码复用率**：与证件照工坊共用 40% 基础框架
**上线周期**：图像工具箱上线后 2-3 周

---

### App 5：文档处理套件

**定位**：拍照即处理，解决日常文档扫描/识别需求

| 功能模块 | 说明 |
|---|---|
| 文档扫描 | 拍照→边缘检测→透视矫正→增强清晰度 |
| OCR 文字识别 | 图片提取文字，支持复制/导出 |
| PDF 转图片 | PDF 每页导出为高清图片 |
| 二维码生成 | 输入链接/文字生成二维码，支持自定义 Logo |
| 二维码识别 | 图片/相机识别二维码内容 |

**技术核心**：Core Vision Kit + PDF Kit + Canvas
**代码复用率**：与证件照工坊共用 30% 基础框架
**说明**：功能最复杂，OCR 精度依赖系统能力，需要单独评估

## 技术底层公共库规划

5个 App 共用以下底层模块，只需开发一次：

| 公共模块 | 说明 | 用于哪些 App |
|---|---|---|
| ImageProcessor | Image Kit 封装，压缩/裁剪/格式转换 | 全部 |
| CanvasRenderer | Canvas 绘图基础封装 | 全部 |
| FileExporter | fileIo + DocumentViewPicker 封装 | 全部 |
| VisionHelper | Core Vision Kit 人像/人脸封装 | 1/2/3/5 |
| SubscriptionManager | 订阅/付费体系 | 全部 |
| UIComponents | 通用组件库（按钮/弹窗/加载） | 全部 |

## 风险清单

| 风险 | 概率 | 应对 |
|---|---|---|
| Core Vision Kit 精度不达标 | 中 | 证件照先验证，不达标降级为手动抠图 |
| 华为审核政策收紧 | 低 | 每个 App 保证功能完整性，不走捷径 |
| 竞品快速跟进 | 中 | 先发优势 + 持续迭代，护住评分和口碑 |
| 鸿蒙生态增速放缓 | 低 | 风险可控，沉没成本仅时间，无运维费用 |
| OCR/PDF 系统 API 能力不足 | 中 | App 5 最后开发，先评估再决定是否做 |
